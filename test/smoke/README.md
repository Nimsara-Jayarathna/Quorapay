# smoke

## Overview
Fast checks for core system viability after changes. These tests should validate startup, leader availability, a basic transaction path, and ledger readability.

## Contents
- Startup and health-check scenarios.
- Single-transaction success path checks.
- Basic leader/follower behavior validation.
- Minimal ledger read verification.

## Not in Scope
- Long-running stress or soak tests.
- Complex partition/failover chaos experiments.
- UI interaction validation.
