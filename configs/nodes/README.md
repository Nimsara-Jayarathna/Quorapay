# nodes

## Overview
Sample node configuration definitions for multi-instance local runs. Each node uses the same service binary and is configured with `NODE_ID`, `PORT`, and `STORAGE_PATH`.

## Contents
- Node ID and listening port mappings.
- Storage path assignments per node instance.
- Cluster membership and peer address examples.
- Non-secret runtime defaults for demos.

## Not in Scope
- Credentials or production secrets.
- Application code or protocol handlers.
- ZooKeeper server configuration internals.
