# team

## Overview
Team-isolated documentation workspace to reduce edit conflicts. Each member owns one dedicated folder and promotes finalized decisions to shared docs through pull requests.

## Contents
- Member-specific design drafts.
- Isolated work logs and investigation notes.
- Branch-aligned technical proposals.
- Handoff notes before promoting to shared docs.

## Not in Scope
- Editing another member's folder contents.
- Canonical final architecture documents.
- Source code or test scripts.

## Ownership
Owner: shared | Branch: shared
